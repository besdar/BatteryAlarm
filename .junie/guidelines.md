# Project Guidelines (BatteryAlarm)

### Code style & documentation expectations (project-specific)
- This is a **learning-focused Kotlin/Android project**. When you make changes:
  - **Be extremely descriptive**: add **as many comments as practical** to explain reasoning and intent.
  - **Document your changes** in the `docs/` folder. Create or update a markdown file that explains **what you did and why**.
  - Favor **clear naming** and **readable structure** over terse code.
