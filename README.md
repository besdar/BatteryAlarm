# Battery Alarm

An Android app that monitors battery levels in real time and alerts the user when the
battery crosses configurable low or high thresholds. A persistent **foreground service**
ensures monitoring continues even when the app is closed, and a **boot receiver**
automatically restarts the service after a device reboot.

## Overview

Battery Alarm is a learning-focused Android project built with **Kotlin** and
**Jetpack Compose**. It demonstrates:

- Foreground services for long-running background work
- Broadcast receivers for system events (`BOOT_COMPLETED`, `ACTION_BATTERY_CHANGED`)
- SharedPreferences for lightweight persistence via a Repository pattern
- Jetpack Compose UI with Material 3, animations, and accessibility support
- Unit testing with **JUnit 4** and **Robolectric**

The user enables battery monitoring from the main screen. The app then runs a foreground
service that listens for battery-level changes and triggers alerts (notification, sound,
vibration) when the level drops below a low threshold or rises above a high threshold.

---

## Features

| Feature | Description |
|---|---|
| **Battery monitoring** | Foreground service listens for `ACTION_BATTERY_CHANGED` broadcasts |
| **Progressive alerts** | Alerts intensify as battery moves further past the threshold |
| **Rate limiting** | Minimum 10 min between alerts (configurable); 30 min for full-charge reminders |
| **Configurable settings** | Low/high thresholds, alert interval, duration, sound, vibration |
| **Reboot survival** | `BootCompletedReceiver` restarts the service after device reboot |
| **Notification permission** | Runtime permission handling for Android 13+ (`POST_NOTIFICATIONS`) |
| **Accessibility** | Content descriptions for all interactive elements |

## Setup & Run

```bash
# 1. Clone the repository
git clone <repository-url>
cd BatteryAlarm

# 2. (Optional) Verify the Gradle wrapper
./gradlew --version

# 3. Build a debug APK
./gradlew assembleDebug

# 4. Install on a connected device / emulator
./gradlew installDebug

# 5. Launch the app
adb shell am start -n com.example.battery_alarm/.MainActivity
```

> **Note:** On first launch the app will request **notification permission** (Android 13+).
> Grant it so the foreground-service notification and battery alerts can appear.

## Scripts / Gradle Tasks

All commands use the Gradle wrapper (`./gradlew` on macOS/Linux, `gradlew.bat` on
Windows).

| Task | Description |
|---|---|
| `./gradlew assembleDebug` | Build debug APK |
| `./gradlew assembleRelease` | Build release APK (unsigned unless signing is configured) |
| `./gradlew installDebug` | Build + install debug APK on connected device |
| `./gradlew testDebugUnitTest` | Run all local (JVM) unit tests |
| `./gradlew connectedDebugAndroidTest` | Run instrumented tests on a device/emulator |
| `./gradlew lint` | Run Android lint checks |
| `./gradlew clean` | Delete all build outputs |

## Tests

The project contains **~59 local unit tests** and one example instrumented test.

### Local Unit Tests (JVM)

| Test File | Class Under Test | Tests | Robolectric? |
|---|---|---|---|
| `SettingsRepositoryTest.kt` | `SettingsRepository` | ~40 | ✅ Yes |
| `BootCompletedReceiverTest.kt` | `BootCompletedReceiver` | 7 | ✅ Yes |
| `BatteryMonitorServiceTest.kt` | `BatteryMonitorService` | 12 | ❌ No |
| `ExampleUnitTest.kt` | — | 1 | ❌ No |

Run them with:

```bash
# All unit tests
./gradlew testDebugUnitTest

# A specific test class
./gradlew testDebugUnitTest --tests "com.example.battery_alarm.data.SettingsRepositoryTest"
```

HTML test report: `app/build/reports/tests/testDebugUnitTest/index.html`

### Instrumented Tests (on-device)


For detailed information on the testing strategy, Robolectric usage, and naming
